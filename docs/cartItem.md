
# 1. 장바구니 페이지 만들기 
- 사용자의 장바구니 리스트 화면

# 2. 디자인
- Pencil MCP 에서 노드 ID가 Nivxk 인 것을 참고(Cart Page)
- Pencil MCP 연동이 없을 경우 구현을 멈춘다.
- 장바구니 리스트는 paging 처리 없음
- 리스트가 화면 height 를 넘길 경우 리스트 영역에 scroll 처리 


# 3. 기능
- 리스트 전체 선택/해제 가능
- 항목별 삭제가 가능
- 수량변경 시 데이터베이스도 바로 업데이트 
- 삭제 시에도 데이터베이스 바로 업데이트 
- 구매하기 버튼은 차후 구현. 디자인만 구현
- 쇼핑계속하기 버튼누르면 main 화면으로 이동

# 4. 서버 API 연동

## 4.1 리스트 가져오기
- /api/carts
- GET 방식
- 파라메터는 별도 없음 
- 결과
```
{
   "code" : 200,
   "data" : [
      {
         "itemId" : "장바구니ID",
         "bookId" : "ID",
         "title" : "타이틀",
         "author" :"저자",
         "publisher" : "출판사",
         "salePrice" : "세일가(최종 판매가)",
         "quantity" :  "수량"
      },
      
   ]
}
```

## 4.2 장바구니 아이템 수량 업데이트
- 수량 변경시  서버 api 요청으로 업데이트 처리 
- /api/carts
- PUT 방식 
- 파라메터 :  { "itemId" :"카트아이템ID",  "quantity" :  "수량" }
- 결과 : { "code" :200 ,"message" : "수량 변경 완료" }
- 코드 값이 200 이 아니면 오류 
- 오류일 경우는 message 를 alert 으로 노출 


## 4.3 장바구니 아이템 삭제
- 삭제 시 서버 api 요청으로 삭제 처리
- 삭제 전에 "정말 삭제하시겠습니까?"  출력 후 OK 하면 삭제 진행 
- /api/carts?items=1,2,3
- DELETE 방식
- 결과 : { "code" :200 ,"message" : "삭제 완료" }
- 코드 값이 200 이 아니면 오류
- 오류일 경우는 message 를 alert 으로 노출
