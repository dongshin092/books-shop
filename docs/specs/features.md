# 메인페이지 기능 명세

## 1. 사용자 스토리 & 수용 기준

### US-01 비로그인 메인페이지 접근
**사용자 스토리**
로그인하지 않은 방문자도 메인페이지에 접근하여 도서 목록을 열람할 수 있다.

**수용 기준**
- 인증 토큰 없이 메인페이지 URL(`/`)에 접근할 수 있다.
- 인증이 없어도 5개 섹션의 도서 카드가 모두 정상 렌더링된다.
- 로그인 상태와 무관하게 동일한 도서 데이터가 노출된다.

---

### US-02 베스트셀러 Top5 열람
**사용자 스토리**
방문자는 메인페이지에서 베스트셀러 도서 Top5를 확인할 수 있다.

**수용 기준**
- 베스트셀러 섹션이 메인페이지에 표시된다.
- `books.best_yn = 'Y'`인 도서만 노출된다.
- 출판일(`published_date`) 내림차순으로 정렬하며, 출판일이 동일하면 제목(`title`) 오름차순으로 정렬한다.
- 최대 5건만 표시된다.
- 각 도서 카드에 표지 이미지(`cover_image`), 제목(`title`), 저자(`author`), 판매가(`sale_price`)가 표시된다.

---

## 2. 도메인 / 데이터 요구사항

### 2.1 5개 섹션 정의

| 섹션명 | 추출 조건 | 데이터 출처 |
|---|---|---|
| 베스트셀러 | `books.best_yn = 'Y'` | books 테이블 |
| 신간 | `books.new_yn = 'Y'` | books 테이블 |
| IT 서적 | `categories.types = 'IT'`인 카테고리에 속한 도서 | books + categories 테이블 |
| 소설 | `categories.types = 'NOVEL'`인 카테고리에 속한 도서 | books + categories 테이블 |
| 자기계발도서 | `categories.types = 'SELF'`인 카테고리에 속한 도서 | books + categories 테이블 |

### 2.2 정렬 규칙
- 1순위: `books.published_date` 내림차순 (최신 출판일 우선)
- 2순위: `books.title` 오름차순 (출판일 동일한 경우 제목 가나다순)

### 2.3 Top5 제한
- 각 섹션당 최대 5건만 반환한다.

### 2.4 도서 카드 표시 필드
각 도서 카드에 다음 4개 필드를 표시한다.

| 필드 | DB 컬럼 | 비고 |
|---|---|---|
| 표지 이미지 | `books.cover_image` | 교보문고 CDN URL 형식 |
| 제목 | `books.title` | |
| 저자 | `books.author` | |
| 판매가 | `books.sale_price` | 정수(원 단위) |

### 2.5 테이블 참조 정보
- **categories**: `id`, `name`, `types`(`IT`/`NOVEL`/`SELF`), `use_yn`
- **books**: `id`, `category_id`(FK), `title`, `author`, `cover_image`, `sale_price`, `published_date`, `best_yn`(`Y`/`N`), `new_yn`(`Y`/`N`)
- 카테고리 타입 비교는 `types` 컬럼의 문자열 값(`'IT'`, `'NOVEL'`, `'SELF'`)으로 한다.

---

## 3. 비기능 요구사항

| 항목 | 내용 |
|---|---|
| 인증 | 메인페이지 도서 데이터 조회는 인증(로그인) 없이 접근 가능해야 한다. |
| 데이터 연동 | 프론트엔드는 백엔드 API를 통해 데이터를 가져온다. 하드코딩 금지. |
| 범위 | 문서에 명시된 5개 섹션 × Top5 이외의 기능은 구현하지 않는다. |

# 도서 상세페이지 기능 명세

## 1. 사용자 스토리 & 수용 기준

### US-D01 도서 상세페이지 진입
**사용자 스토리**
방문자(로그인 여부 무관)는 어떤 페이지에서든 도서 제목(또는 표지)을 클릭하면 해당 도서의 상세페이지로 이동할 수 있다.

**수용 기준**
- 도서 카드의 제목 또는 표지 이미지를 클릭하면 `/books/{bookId}` 경로로 이동한다.
- 비로그인 상태에서도 상세페이지에 접근할 수 있다.
- 유효하지 않은 `bookId`로 접근 시 적절한 에러 메시지를 표시한다.

---

### US-D02 도서 상세정보 조회
**사용자 스토리**
방문자는 도서 상세페이지에서 해당 도서의 상세정보를 확인할 수 있다.

**수용 기준**
- 페이지 진입 시 백엔드 API를 통해 도서 상세 데이터를 가져온다.
- 다음 항목이 모두 화면에 표시된다: 표지 이미지, 제목, 저자, 출판사, 출간일, 정가(listPrice), 판매가(salePrice), 책 소개(description).
- 정가와 판매가가 다를 경우 할인율을 함께 표시한다(디자인 기준: `15% 할인` 형태).
- 데이터 로딩 중에는 로딩 상태를 표시한다.

---

### US-D03 독자 리뷰 목록 조회 및 페이지네이션
**사용자 스토리**
방문자는 도서 상세페이지에서 독자 리뷰 목록을 확인하고, 페이지 단위로 탐색할 수 있다.

**수용 기준**
- '독자 리뷰' 섹션에 총 리뷰 개수(예: `총 48개`)가 표시된다.
- 리뷰 목록은 한 페이지에 `size`건씩 표시된다.
- 각 리뷰 항목에 작성자명, 작성일(YYYY.MM.DD 형식), 리뷰 내용, 별점(0~5점, 별 아이콘)이 표시된다.
- 페이지네이션 UI(이전/다음/번호)가 표시되며, 현재 페이지가 강조 표시된다.
- 리뷰가 없는 경우 빈 목록(0건)으로 처리하며 에러를 발생시키지 않는다.
- 리뷰 데이터는 도서 상세 API(`GET /api/books/{bookId}`) 응답의 `reviewList`로 제공된다.

---

### US-D04 장바구니 담기
**사용자 스토리**
로그인한 사용자는 도서 상세페이지에서 '장바구니' 버튼을 눌러 해당 도서를 장바구니에 담을 수 있다.

**수용 기준**
- 로그인 상태에서 '장바구니' 버튼을 누르면 장바구니 담기 동작이 실행된다.
- 비로그인 상태에서 '장바구니' 버튼을 누르면 `'로그인이 필요한 기능입니다'` 경고 팝업을 표시하고, 장바구니 담기 동작은 실행하지 않는다.

---

### US-D05 바로구매
**사용자 스토리**
로그인한 사용자는 도서 상세페이지에서 '바로구매' 버튼을 눌러 즉시 구매 절차를 진행할 수 있다.

**수용 기준**
- 로그인 상태에서 '바로구매' 버튼을 누르면 구매 절차가 실행된다.
- 비로그인 상태에서 '바로구매' 버튼을 누르면 `'로그인이 필요한 기능입니다'` 경고 팝업을 표시하고, 구매 절차는 실행하지 않는다.

## 2. 도메인 / 데이터 요구사항

### 2.1 도서 상세 필드

| 필드명 | 설명 | 비고 |
|---|---|---|
| `bookId` | 도서 고유 식별자 | 경로 변수(`{bookId}`) |
| `title` | 도서 제목 | |
| `coverImage` | 표지 이미지 경로 | CDN URL 형식 |
| `author` | 저자 | |
| `listPrice` | 정가 | 정수(원 단위) |
| `salePrice` | 판매가(최종 결제액) | 정수(원 단위) |
| `publisher` | 출판사 | |
| `stocks` | 재고 수량 | 정수 |
| `publishedDate` | 출간일 | |
| `description` | 책 소개 | |
| `reviewList` | 독자 리뷰 목록 | 페이지네이션 적용 |

### 2.2 리뷰(독자 리뷰) 필드

디자인(`book-detail-page.png`) 기준으로 각 리뷰 항목에 노출되는 필드는 다음과 같다.

| 필드명 | 설명 | 비고 |
|---|---|---|
| 작성자명 | 리뷰 작성자 이름 | 아바타(성 첫 글자) 함께 표시 |
| 작성일 | 리뷰 작성 날짜 | `YYYY.MM.DD` 형식으로 표시 |
| 별점 | 0~5점 사이 정수 또는 반 단계 | 별 아이콘(★/☆)으로 시각화 |
| 내용 | 리뷰 본문 텍스트 | |

### 2.3 리뷰 페이징 및 정렬 규칙

- 리뷰는 API의 `page`(0부터 시작), `size`(한 페이지에 보여줄 개수) 파라미터로 페이징한다.
- 정렬 기준: **최신순**(작성일 내림차순)을 기본으로 한다.
- 응답에 총 리뷰 개수(`totalCount` 또는 동급 필드)를 포함하여 프론트에서 페이지네이션 UI를 구성할 수 있게 한다.
- 리뷰 데이터가 없는 경우 빈 배열(`[]`)을 반환하며 오류 처리를 하지 않는다.

### 2.4 API 연동 방식

- 엔드포인트: `GET /api/books/{bookId}?page={page}&size={size}`
- 도서 상세정보와 해당 페이지의 리뷰 목록을 단일 응답으로 반환한다(`reviewList` 포함).
- Review 물리 DB 테이블 및 시드 데이터는 이 단계에서 생성하지 않는다. 백엔드는 엔티티 및 조회 코드만 작성하며, 데이터 없으면 빈 목록을 반환한다.

# 카테고리별 도서 리스트 기능 명세

## 1. 사용자 스토리 & 수용 기준

### US-L01 카테고리 메뉴로 도서 목록 진입
**사용자 스토리**
방문자(로그인 여부 무관)는 전역 메뉴에서 IT/소설/자기개발 카테고리를 선택하여 해당 카테고리의 도서 목록 화면으로 이동할 수 있다.

**수용 기준**
- 헤더 메뉴에서 카테고리(IT / 소설 / 자기개발)를 선택하면 해당 카테고리의 도서 목록 화면으로 이동한다.
- 인증 토큰 없이도 목록 화면에 접근할 수 있다.
- 화면 진입 시 백엔드 API를 통해 해당 카테고리의 도서 목록을 가져온다(mock 데이터 미사용).

---

### US-L02 카테고리별 도서 목록 열람
**사용자 스토리**
방문자는 선택한 카테고리에 속한 도서 목록을 확인할 수 있다.

**수용 기준**
- `categories.types`가 선택한 카테고리 값(`IT` / `NOVEL` / `SELF`)과 일치하는 도서만 노출된다.
- 목록에 해당 카테고리의 전체 도서 건수(총 N권)가 표시된다.
- 도서가 없는 카테고리는 빈 목록으로 표시하며 오류를 발생시키지 않는다.
- 각 도서 항목에 표지, 제목, 부제목, 저자, 출판사, 출판일, 평점, 리뷰수, 정가, 판매가가 표시된다(필드 상세는 2.4 참조).

---

### US-L03 도서 목록 기본 정렬(최신순)
**사용자 스토리**
방문자는 목록 진입 시 기본적으로 최신 출판일 순으로 정렬된 도서를 본다.

**수용 기준**
- 화면 최초 진입 시 정렬 기준은 "최신순"(`orderType=new`)이다.
- `publishedDate` 내림차순으로 정렬한다.
- 출판일이 동일한 도서는 `title` 오름차순으로 2차 정렬한다.

---

### US-L04 정렬 옵션 변경
**사용자 스토리**
방문자는 목록 화면에서 정렬 기준을 최신순/낮은가격순/높은가격순/리뷰많은순 중에서 선택하여 바꿀 수 있다.

**수용 기준**
- 정렬 탭에 4가지 옵션(최신순/낮은가격순/높은가격순/리뷰많은순)이 표시된다.
- 낮은가격순(`orderType=lower`) 선택 시 `salePrice` 오름차순으로 정렬한다.
- 높은가격순(`orderType=high`) 선택 시 `salePrice` 내림차순으로 정렬한다.
- 리뷰많은순(`orderType=reviewCnt`) 선택 시 리뷰 수(`totalReviewCnt`) 내림차순으로 정렬한다.
- 정렬 기준을 변경하면 페이지는 1페이지(첫 페이지)로 초기화된다.
- 현재 선택된 정렬 옵션은 시각적으로 강조 표시된다.

### US-L05 도서 목록 페이징
**사용자 스토리**
방문자는 도서 목록이 많을 경우 페이지 단위로 나누어 탐색할 수 있다.

**수용 기준**
- 도서 목록은 페이지당 정해진 건수만큼 표시된다.
- 페이지네이션 UI(처음/이전/번호/다음/끝)로 다른 페이지를 탐색할 수 있다.
- 현재 페이지가 시각적으로 강조 표시된다.
- 페이지를 이동하면 화면 상단으로 스크롤된다.
- 정렬 기준은 페이지 이동과 무관하게 유지된다.

---

### US-L06 도서 상세로 이동
**사용자 스토리**
방문자는 목록에서 도서 제목(또는 표지)을 클릭하여 해당 도서의 상세페이지로 이동할 수 있다.

**수용 기준**
- 도서 제목 또는 표지를 클릭하면 해당 도서의 상세페이지(`/books/{bookId}`)로 이동한다.

---

### US-L07 장바구니 담기 / 바로구매 (비로그인 제어)
**사용자 스토리**
로그인한 사용자는 목록에서 바로 장바구니에 담거나 즉시 구매를 진행할 수 있다.

**수용 기준**
- 로그인 상태에서 '장바구니' 또는 '바로구매' 버튼을 누르면 해당 동작이 실행된다.
- 비로그인 상태에서 두 버튼 중 하나를 누르면 `'로그인이 필요한 기능입니다'` 경고 팝업을 표시하고, 실제 동작은 실행하지 않는다.

## 2. 도메인 / 데이터 요구사항

### 2.1 카테고리 타입

| 값 | 의미 |
|---|---|
| `IT` | IT 서적 |
| `NOVEL` | 소설 |
| `SELF` | 자기개발서 |

카테고리 구분은 `categories.types` 값을 기준으로 하며, 대상 도서는 `books.category_id`로 `categories`와 조인하여 판별한다.

### 2.2 정렬 규칙(orderType)

| orderType 값 | 의미 | 정렬 기준 |
|---|---|---|
| `new` (기본값) | 최신순 | `publishedDate` 내림차순, 동일 시 `title` 오름차순 |
| `lower` | 낮은가격순 | `salePrice` 오름차순 |
| `high` | 높은가격순 | `salePrice` 내림차순 |
| `reviewCnt` | 리뷰많은순 | 리뷰 수 내림차순 |

- 가격 정렬은 판매가(`salePrice`) 기준으로 한다(정가 기준 아님).

### 2.3 페이징

- 목록은 페이지 단위로 조회하며, 페이지 번호와 페이지당 건수를 파라미터로 받는다.
- 카테고리별 전체 건수를 함께 제공하여 화면에 총 건수와 페이지네이션을 구성할 수 있게 한다.

### 2.4 화면 표시 필드

| 필드 | 설명 | 비고 |
|---|---|---|
| 제목 | 도서 제목 | |
| 부제목 | 도서 부제목 | |
| 저자 | 저자명 | |
| 출판사 | 출판사명 | |
| 출판일 | 출판일 | |
| 평점 | 리뷰 평균 점수 | 0~10점 척도(2.5 참조) |
| 리뷰수 | 전체 리뷰 개수 | |
| 정가 | 할인 전 가격 | 정가와 판매가가 다르면 할인율 함께 표시 |
| 판매가 | 실제 판매 가격 | |
| 표지 | 표지 이미지 | |
| 뱃지 | 베스트/신간 등 도서 특성 표시 | 2.6 참조 |

### 2.5 평점(reviewRating) 척도

- 목록 API가 반환하는 `reviewRating`은 **0~10점 척도**의 평균값이다.
- 화면은 기존 방식대로 이 값을 2로 나누어(`/2`) 5개 별 아이콘으로 환산해 표시한다.

### 2.6 뱃지(BEST/NEW) 표시 규칙

- 목록 화면은 도서별로 `BEST` 또는 `NEW` 뱃지를 표시하며, 이 필드는 목록 API 응답에 **포함**된다.
- 메인페이지 명세(US-02, US-03)의 `books.best_yn`, `books.new_yn` 플래그를 기준으로 판별한다.

### 2.7 부제목(subtitle) 필드

- 목록 API 응답에 `subtitle`(부제목)을 **포함**한다.

## 3. 비기능 요구사항

| 항목 | 내용 |
|---|---|
| 인증 | 목록 조회는 인증(로그인) 없이 접근 가능해야 한다. |

| 페이지 크기(size) 기본값 | 파라미터 미지정 시 기본값은 **10**이다. |

## 4. 팀장 확인 결과 (확정)

1. **평점 척도**: `reviewRating`은 **0~10점** 척도로 확정. 화면은 기존대로 `/2` 환산하여 별 아이콘 표시.
2. **부제목(subtitle)**: 목록 API 응답에 **포함**.
3. **뱃지(BEST/NEW)**: 목록 API 응답에 **포함**. `books.best_yn` / `books.new_yn` 기준으로 판별.
4. **페이지 크기(size) 기본값**: **10**으로 확정.
