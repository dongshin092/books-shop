# 메인페이지 기능 명세

## 1. 사용자 스토리 & 수용 기준

### US-01 비로그인 메인페이지 접근
**사용자 스토리**
로그인하지 않은 방문자도 메인페이지에 접근하여 도서 목록을 열람할 수 있다.

**수용 기준**
- 인증 토큰 없이 메인페이지 URL(`/`)에 접근할 수 있다.
- 인증이 없어도 5개 섹션의 도서 카드가 모두 정상 렌더링된다.
- 로그인 상태와 무관하게 동일한 도서 데이터가 노출된다.

---

### US-02 베스트셀러 Top5 열람
**사용자 스토리**
방문자는 메인페이지에서 베스트셀러 도서 Top5를 확인할 수 있다.

**수용 기준**
- 베스트셀러 섹션이 메인페이지에 표시된다.
- `books.best_yn = 'Y'`인 도서만 노출된다.
- 출판일(`published_date`) 내림차순으로 정렬하며, 출판일이 동일하면 제목(`title`) 오름차순으로 정렬한다.
- 최대 5건만 표시된다.
- 각 도서 카드에 표지 이미지(`cover_image`), 제목(`title`), 저자(`author`), 판매가(`sale_price`)가 표시된다.

## 2. 도메인 / 데이터 요구사항

### 2.1 5개 섹션 정의

| 섹션명 | 추출 조건 | 데이터 출처 |
|---|---|---|
| 베스트셀러 | `books.best_yn = 'Y'` | books 테이블 |
| 신간 | `books.new_yn = 'Y'` | books 테이블 |
| IT 서적 | `categories.types = 'IT'`인 카테고리에 속한 도서 | books + categories 테이블 |
| 소설 | `categories.types = 'NOVEL'`인 카테고리에 속한 도서 | books + categories 테이블 |
| 자기계발도서 | `categories.types = 'SELF'`인 카테고리에 속한 도서 | books + categories 테이블 |

### 2.2 정렬 규칙
- 1순위: `books.published_date` 내림차순 (최신 출판일 우선)
- 2순위: `books.title` 오름차순 (출판일 동일한 경우 제목 가나다순)

### 2.3 Top5 제한
- 각 섹션당 최대 5건만 반환한다.

### 2.4 도서 카드 표시 필드
각 도서 카드에 다음 4개 필드를 표시한다.

| 필드 | DB 컬럼 | 비고 |
|---|---|---|
| 표지 이미지 | `books.cover_image` | 교보문고 CDN URL 형식 |
| 제목 | `books.title` | |
| 저자 | `books.author` | |
| 판매가 | `books.sale_price` | 정수(원 단위) |

### 2.5 테이블 참조 정보
- **categories**: `id`, `name`, `types`(`IT`/`NOVEL`/`SELF`), `use_yn`
- **books**: `id`, `category_id`(FK), `title`, `author`, `cover_image`, `sale_price`, `published_date`, `best_yn`(`Y`/`N`), `new_yn`(`Y`/`N`)
- 카테고리 타입 비교는 `types` 컬럼의 문자열 값(`'IT'`, `'NOVEL'`, `'SELF'`)으로 한다.

---

## 3. 비기능 요구사항

| 항목 | 내용 |
|---|---|
| 인증 | 메인페이지 도서 데이터 조회는 인증(로그인) 없이 접근 가능해야 한다. |
| 데이터 연동 | 프론트엔드는 백엔드 API를 통해 데이터를 가져온다. 하드코딩 금지. |
| 범위 | 문서에 명시된 5개 섹션 × Top5 이외의 기능은 구현하지 않는다. |

# 도서 상세페이지 기능 명세

## 1. 사용자 스토리 & 수용 기준

### US-D01 도서 상세페이지 진입
**사용자 스토리**
방문자(로그인 여부 무관)는 어떤 페이지에서든 도서 제목(또는 표지)을 클릭하면 해당 도서의 상세페이지로 이동할 수 있다.

**수용 기준**
- 도서 카드의 제목 또는 표지 이미지를 클릭하면 `/books/{bookId}` 경로로 이동한다.
- 비로그인 상태에서도 상세페이지에 접근할 수 있다.
- 유효하지 않은 `bookId`로 접근 시 적절한 에러 메시지를 표시한다.

---

### US-D02 도서 상세정보 조회
**사용자 스토리**
방문자는 도서 상세페이지에서 해당 도서의 상세정보를 확인할 수 있다.

**수용 기준**
- 페이지 진입 시 백엔드 API를 통해 도서 상세 데이터를 가져온다.
- 다음 항목이 모두 화면에 표시된다: 표지 이미지, 제목, 저자, 출판사, 출간일, 정가(listPrice), 판매가(salePrice), 책 소개(description).
- 정가와 판매가가 다를 경우 할인율을 함께 표시한다(디자인 기준: `15% 할인` 형태).
- 데이터 로딩 중에는 로딩 상태를 표시한다.

---

### US-D03 독자 리뷰 목록 조회 및 페이지네이션
**사용자 스토리**
방문자는 도서 상세페이지에서 독자 리뷰 목록을 확인하고, 페이지 단위로 탐색할 수 있다.

**수용 기준**
- '독자 리뷰' 섹션에 총 리뷰 개수(예: `총 48개`)가 표시된다.
- 리뷰 목록은 한 페이지에 `size`건씩 표시된다.
- 각 리뷰 항목에 작성자명, 작성일(YYYY.MM.DD 형식), 리뷰 내용, 별점(0~5점, 별 아이콘)이 표시된다.
- 페이지네이션 UI(이전/다음/번호)가 표시되며, 현재 페이지가 강조 표시된다.
- 리뷰가 없는 경우 빈 목록(0건)으로 처리하며 에러를 발생시키지 않는다.
- 리뷰 데이터는 도서 상세 API(`GET /api/books/{bookId}`) 응답의 `reviewList`로 제공된다.

---

### US-D04 장바구니 담기
**사용자 스토리**
로그인한 사용자는 도서 상세페이지에서 '장바구니' 버튼을 눌러 해당 도서를 장바구니에 담을 수 있다.

**수용 기준**
- 로그인 상태에서 '장바구니' 버튼을 누르면 장바구니 담기 동작이 실행된다.
- 비로그인 상태에서 '장바구니' 버튼을 누르면 `'로그인이 필요한 기능입니다'` 경고 팝업을 표시하고, 장바구니 담기 동작은 실행하지 않는다.

---

### US-D05 바로구매
**사용자 스토리**
로그인한 사용자는 도서 상세페이지에서 '바로구매' 버튼을 눌러 즉시 구매 절차를 진행할 수 있다.

**수용 기준**
- 로그인 상태에서 '바로구매' 버튼을 누르면 구매 절차가 실행된다.
- 비로그인 상태에서 '바로구매' 버튼을 누르면 `'로그인이 필요한 기능입니다'` 경고 팝업을 표시하고, 구매 절차는 실행하지 않는다.

## 2. 도메인 / 데이터 요구사항

### 2.1 도서 상세 필드

| 필드명 | 설명 | 비고 |
|---|---|---|
| `bookId` | 도서 고유 식별자 | 경로 변수(`{bookId}`) |
| `title` | 도서 제목 | |
| `coverImage` | 표지 이미지 경로 | CDN URL 형식 |
| `author` | 저자 | |
| `listPrice` | 정가 | 정수(원 단위) |
| `salePrice` | 판매가(최종 결제액) | 정수(원 단위) |
| `publisher` | 출판사 | |
| `stocks` | 재고 수량 | 정수 |
| `publishedDate` | 출간일 | |
| `description` | 책 소개 | |
| `reviewList` | 독자 리뷰 목록 | 페이지네이션 적용 |

### 2.2 리뷰(독자 리뷰) 필드

디자인(`book-detail-page.png`) 기준으로 각 리뷰 항목에 노출되는 필드는 다음과 같다.

| 필드명 | 설명 | 비고 |
|---|---|---|
| 작성자명 | 리뷰 작성자 이름 | 아바타(성 첫 글자) 함께 표시 |
| 작성일 | 리뷰 작성 날짜 | `YYYY.MM.DD` 형식으로 표시 |
| 별점 | 0~5점 사이 정수 또는 반 단계 | 별 아이콘(★/☆)으로 시각화 |
| 내용 | 리뷰 본문 텍스트 | |

### 2.3 리뷰 페이징 및 정렬 규칙

- 리뷰는 API의 `page`(0부터 시작), `size`(한 페이지에 보여줄 개수) 파라미터로 페이징한다.
- 정렬 기준: **최신순**(작성일 내림차순)을 기본으로 한다.
- 응답에 총 리뷰 개수(`totalCount` 또는 동급 필드)를 포함하여 프론트에서 페이지네이션 UI를 구성할 수 있게 한다.
- 리뷰 데이터가 없는 경우 빈 배열(`[]`)을 반환하며 오류 처리를 하지 않는다.

### 2.4 API 연동 방식

- 엔드포인트: `GET /api/books/{bookId}?page={page}&size={size}`
- 도서 상세정보와 해당 페이지의 리뷰 목록을 단일 응답으로 반환한다(`reviewList` 포함).
- Review 물리 DB 테이블 및 시드 데이터는 이 단계에서 생성하지 않는다. 백엔드는 엔티티 및 조회 코드만 작성하며, 데이터 없으면 빈 목록을 반환한다.

## 3. 비즈니스 규칙

| 규칙 | 내용 |
|---|---|
| 비로그인 진입 | 로그인 여부와 무관하게 도서 상세페이지 및 리뷰 목록 조회가 가능하다. |
| 장바구니/바로구매 접근 제어 | 비로그인 상태에서 장바구니 또는 바로구매 버튼 클릭 시 `'로그인이 필요한 기능입니다'` 경고 팝업을 표시한다. |

| 범위 | 리뷰 작성/수정/삭제 기능은 이번 범위에 포함하지 않는다(조회만). |
