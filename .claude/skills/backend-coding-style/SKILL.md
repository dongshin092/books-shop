# 백엔드 코드 스타일 규약

코드를 작성하거나 수정할 때 아래 스타일을 따른다.

## 주석 규칙

- **주석은 이해하기 어려운 코드에만 단다.** 코드만으로 의도가 분명하면 주석을 달지 않는다.
- 코드를 그대로 한국어로 옮긴 수준의 주석은 금지한다.

  ```java
  // 나쁨: 코드와 동일한 내용
  // id로 사용자를 조회한다
  User user = userRepository.findById(id).orElseThrow(...);

  // 좋음: 왜(이유)를 설명
  // 외부 결제사 규격상 금액이 음수로 올 수 있어 절댓값으로 보정한다
  long amount = Math.abs(rawAmount);
  ```

- 주석으로 설명해야 할 만큼 복잡하면, 먼저 이름·구조 개선으로 주석이 필요 없게 만들 수 있는지 검토한다.
- 공개 API나 복잡한 도메인 규칙에는 필요한 경우 간결한 Javadoc 또는 설명 주석을 허용한다.

## 일반 스타일

- 의미 있는 이름을 사용한다. 변수·메서드는 역할이 드러나게 짓는다.
- 메서드는 한 가지 일만 하도록 작게 유지한다.
- 매직 넘버·문자열은 상수로 추출한다.
